# 12 — Project Structure

## Repo layout

```
OmniCaption/
├── AGENTS.md                 # rules every assistant follows (shared state)
├── STATUS.md                 # live board: who is doing what (shared state)
├── CLAUDE.md                 # Claude entry point (Tumo)
├── GEMINI.md                 # Gemini entry point (Katlego)
├── Hackathon Research for AI Video Captioning.pdf   # source research
├── docs/                     # this planning documentation set (00–14 + deployment)
├── specs/                    # Spec-Kit specifications
│   ├── plan.md               # technical plan (from `plan`)
│   └── tasks.md              # checkbox task list, task ids like T012 (from `tasks`)
├── .specify/                 # Spec-Kit scaffold — GENERATED by `specify init`
│   └── scripts/              #   (do not hand-edit; re-run tooling)
├── .bob/                     # IBM Bob working dir + manifests — GENERATED
├── .githooks/                # pre-push test gate (wired via core.hooksPath)
├── services/
│   └── captioner/            # the Python pipeline (the product)
│       ├── ingestion.py      # Stage 1: tasks.json, download, ffmpeg WAV
│       ├── audio.py          # Stage 2: faster-whisper HIP transcription
│       ├── memory.py         # Stage 3: VRAM reclamation
│       ├── vision.py         # Stage 4: OpenCV keyframes + alignment
│       ├── synthesis.py      # Stage 5: Gemma 4 E4B, modality order, PMP
│       ├── output.py         # Stage 6: schema validation, results.json
│       ├── prompts/          # the four style system prompts
│       ├── config.py         # pydantic-settings config
│       └── main.py           # entry point: runs stages 1→6, exits 0
├── apps/
│   └── web/                  # OPTIONAL Track-3 "Video-Oracle" demo UI
├── Dockerfile                # linux/amd64 build
└── tests/                    # pytest: contract, golden-clip, latency
```

Generated vs. human-seeded artifacts (`.bob/`, `.specify/`) are explained in
[07-ibm-bob-spec-kit](07-ibm-bob-spec-kit.md). Shared-state coordination is in
[10-cross-ai-protocol](10-cross-ai-protocol.md).

## How to run each part

### The captioner pipeline (Track 2)

```bash
docker run --rm -v ./input:/input -v ./output:/output <image>
```

Runs `services/captioner/main.py` through stages 1→6 and writes `/output/results.json`. Local build
and smoke-test details are in [deployment](deployment.md).

### Tests

```bash
pytest            # contract + golden-clip + latency (models mocked)
ruff check .      # lint, line length 100
```

See [04-testing-strategy](04-testing-strategy.md).

### The web demo (Track 3, optional)

`apps/web/` hosts the "Video-Oracle" semantic-search demo. It is a stretch deliverable and the first
thing cut under time pressure ([00-project-plan](00-project-plan.md)); run it only when Track 2 is
green.
